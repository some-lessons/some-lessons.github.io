Learning goals:

1. Be able to state where data is stored in a typical web application.

2. Be able to state what kinds of devices send HTTP requests, and why.

3. Be able to state what someone might do to scale a web app in the case that the backend has more requests than it can handle, and if there are any requirements of the web application to be scaled this way.

4. Be able to draw a diagram of a load-balanced web application.

Estimated time: 20 minutes.

Recommendation:

1. Visit https://gist.github.com/vasanthk/485d1c25737e8e72759f and look at the diagram.

2. Read https://ruben.verborgh.org/blog/2012/08/24/rest-wheres-my-state/

3. Discuss the learning goals with the instructor.
